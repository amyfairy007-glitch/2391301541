# Git 建分支 AI 提示词

> 使用位置：在目标项目仓库根目录打开 AI / OpenCode 后使用。  
> 使用前提：当前终端所在目录就是需要创建分支的项目仓库。

```text
你现在位于目标项目仓库根目录。

我会提供一个 Jira ticket 地址。请按以下规则处理 Git 建分支：

1. 读取 Jira ticket，提取：
   - Jira 编号
   - Jira 中的 Journey 名称

2. 按以下格式生成分支名：
   bugfix/<Jira编号>-<Journey名称>

3. 在执行任何 Git 操作前，先输出以下内容让我核对：

   - Jira 编号
   - 识别到的 Journey 名称
   - 准备创建的分支名
   - 基线：远程最新 master

4. 未经我明确确认，不执行任何 Git 命令。

5. 我确认后：
   - 更新远程 master 引用；
   - 基于远程最新 master 创建新分支；
   - 切换到新分支；
   - 输出：
     - 当前分支名
     - 基线提交号
     - Git 状态

6. 禁止执行以下操作：
   - 不提交代码
   - 不推送代码
   - 不修改代码
   - 不修改现有分支
   - 不删除分支
   - 不合并代码
   - 不重置或清理工作区

7. 若 Jira 中无法明确识别 Journey 名称，不要猜测，不要创建分支；先把你识别到的信息发给我确认。
```

## 实际调用方式

```text
按 Git 建分支规则处理这个 Jira：
<Jira ticket 地址>
```
