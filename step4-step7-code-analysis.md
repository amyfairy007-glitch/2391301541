## 提取统一代码分析规则

请从成熟模板提取规则，至少覆盖：

### 文件层面

- 文件新增、删除、移动、重命名
- 文件类型变化
- 入口文件与配置文件变化
- 新旧文件如何判定为同一业务功能

### 模块与组件层面

- 模块或组件新增、删除
- 组件层级、拆分、合并
- 公共组件复用
- 服务层和数据层变化

### 函数和方法层面

- 函数新增、删除、重命名
- 参数、返回值、调用链变化
- 同步异步、副作用、异常处理变化

### API 层面

- endpoint、method、request、response schema
- header、错误码、重试、超时
- mock 与真实 API 差异

只比较 API 定义和使用方式，不比较运行时客户数据。

### 数据流层面

- 数据来源、转换、映射、过滤、提交、回填
- 状态同步、缓存、本地存储、session、上下游依赖

### 校验规则层面

- required、格式、长度、范围、条件、跨字段校验
- 前后端校验差异
- 错误提示触发逻辑

不得复制成熟 Journey 的具体字段名称。

### 条件分支层面

- if/else、feature flag、权限、状态、用户类型、产品类型
- 环境判断、fallback、默认分支

具体业务条件必须抽象化或配置化。

### 状态管理层面

- state 初始化与更新
- store、reducer/action、context/provider
- 表单状态、loading/error/success

### 错误处理层面

- try/catch、error boundary
- API 错误、表单错误、fallback UI
- 日志、提示、重试、失败恢复

### 依赖和配置层面

- package 依赖与版本
- feature、route、environment、build、API、schema 配置
- 常量和枚举变化
